## 随机算法

重要参数：**控制失败概率**
- Pr【ALG is correct】>=1-δ,correct的定义很多
- 多次运行且要保证全部成功，就让失败概率和次数有关,下届估算:==存在(至少)一次失败的概率小于等于失败概率之和==，多次运行出现一次失败的概率至多nδ

第一步 **数学期望E(x)**
- 期望X的值在Ex附近（不保证）
- 因此一般是要求==Ex是correct的（这是必要条件）==

第二步 **correct概率大于常数**
- 若X>=0,则X>=a的概率<=Ex/a

第三步 多次重复降低失败概率 ,**O（log (1/δ)）次**,δ是失败概率

让数据在**期望附近**的主要要求：1、单边误差且都大于0。2、变量方差比较小。3、x的最坏情况和Ex的比比较小

### 关于随机数

大多时候的随机数生成器使用的是**伪随机**（真随机代价比较大）：通过确定的**递推函数**（都是吗？）来生成

#### 回答给定数据的出现次数
- count-min sketch
	- 将大的值域哈希到小的值域中，这样只需要小的内存即可记录数字，查询的结果大于等于这个数字真实的出现次数。一般查询出现次数比较多的数字会更准确                        
	- 可以设置Ologn个独立哈希，取最小值
	- bucket也要大一些，使期望更接近真实值
	- count-min的计数结果（哈希方式相同）可以相互加减，从而计算多个数组结合的结果
- k-heavy hitter
	- **数据流**：数据只输入一遍，没存就没了
	- 算法：维护一个sketch，每次插入元素，**维护插入次数M**，都用这个sketch查询元素的出现次数，将出现次数（这是一个估计值）>=M/k的元素**存入L**。每次插入后扫描L中的元素，**删除L中次数<M/k的元素**（其实不用着急删）（删除应该是为了防止内存占用太大）

-  数据流精确找 Majority（即出现至少一半次数的元素）。支持动态数据流（插入 / 删除）


	- 如果仅仅找 majority，那么可以用更简单的确定性方法找到精确解
	- 观察：考虑二进制表示，如果一个元素 x 出现超过一半，那么单看所有元素的某一位 t，xt​出现也是过半的
	- 比如{110,110,110,111}，那么在所有 3 位中，110 对应位置的值都出现过半
	- 算法：维护总元素个数c，及对1≤t≤logn，c__t​代表第 t 位 1 的个数

- Boyer-Moore Majority Vote Algorithm
	- 在长度为 `n` 的数组 / 数据流中，出现次数**严格超过 `n/2`** 的元素（注意：不是 “至少一半”，是严格过半）。
	- 元素大小不超过机器字长（比如 32/64 位整数）、且数据流 **只支持插入（不支持删除）**
	- 由于数字是绝对多数。所以抵消后剩下的必定是这个数字（下面是伪代码）
```
counter ← 0, maj ← ⊥ 
for input i 
	if counter = 0 || maj == i 
		maj ← i, counter++ 
	else counter--
 return maj
```

### jaccard similarity
- **定义**：J(A,B)=|AB|/|A+B| if|A+B|>0 else 0    一种度量相似度的方式  如何快速计算？
- 一种基于hash的方法
	- 对每一个集合，预处理一个hash h（各个集合的并集），使得J（A，B）可由hash值得到。**预处理需要O（|A|），查询O（1）
	- 思路：将1到n 随机hash到 \[0,1]（**必须要求每个数字的hash值都不相等**） ,设h_min (A)=min h(x,x in A).可得**Pr（hmin（A）=hmin(B)）=J(A,B)**[^1]（*好秀啊*）
	- **完整算法**：在上面的基础上多次（T次）独立hash实验得到概率
	- 当J非常小时非常难处理（==实际上对于很多采样，在概率非常小时都很难==），J=0什么算法都很难
	- **误差**，在T的数字**固定**时，当J比较小时，由于输出的估计的**分辨率为1/T**,则存在**绝对的误差（不必考虑相对误差）**，当J较大时，则会出现相对误差
		- 有一个算法是：T的大小不固定，当实验的结果都是0时，增加T的大小（或许倍增），直到出现一次结果非0

[^1]: 由于每个数字对应的hash值都可能最小（且只需要考虑A+B），则每个数字hash值最小的概率为1/(A+B),同时每个数字的hash值都不一样，你们两个h_min相等时只有AB中的数字的hash最小时才成立，则概率为|AB|/|A+B|

### hamming近似最近邻查询
目标：对于d维的hamming空间H^d上的点，预处理时间$O(dn+n^{1+\frac{1}{c}}dlogn)$，给定q属于H^d,$O(n^\frac{1}{c}dlogn)$时间找到c近似最近邻（若最近的距离为r，返回的点的距离<=cr）

算法：
- 首先观察到，将坐标随机排列，则距离比较近的前k位大概率相同（如果不随机排列可能会出现极端情况使得前几位不同但是大部分位相同）
- 预处理：
	- 设$N=O(n^{\frac{1}{c}})$,生成N个排列方式O_i,将这些排列方式应用到P（数据集）得到P_i
	- 将得到的P_i按照字典序排序（为了高效搜索最大公共前缀）(这样距离最大公共前缀的数据越远，公共前缀越小)
- 查询：
	- 查询一个q时，将O_i作用到q得到q_i;
	- 找到x_i,为qi在Pi的LCP（最大公共前缀）
	- 将xi对应的一堆数据放入S
	- 暴力搜索q在S上的最近邻
