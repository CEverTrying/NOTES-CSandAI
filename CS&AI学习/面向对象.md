### 继承中的访问权限变化

当你定义一个子类时（如 `class B : public A`），冒号后面的 `public` 也是一种访问修饰符，它决定了**父类的成员在子类中"表现"成什么样**。

这看起来很复杂，但只需要记住这个**降级原则**：

> **继承方式决定了父类成员在子类中的最高权限。**  
> 如果继承方式比成员原本的权限更严格，成员的权限就会被"降级"到继承方式的级别。

```cpp
class ChildA : public Parent{}
class ChildB : protected Parent{}
```
### 构造函数

类的**构造函数**是类的一种特殊的成员函数，它会在每次创建类的新对象时执行。

构造函数的名称与类的名称是完全相同的，并且不会返回任何类型，也不会返回 void。构造函数可用于为某些成员变量设置初始值。

#### 使用初始化列表来初始化字段

使用初始化列表来初始化字段：
```cpp
Line::Line( double len): length(len) 
{
 cout << "Object is being created, length = " << len << endl; 
 }
```


上面的语法等同于如下语法：
```cpp
Line::Line( double len) 
{ length = len;
 cout << "Object is being created, length = " << len << endl;
}
```


假设有一个类 C，具有多个字段 X、Y、Z 等需要进行初始化，同理地，您可以使用上面的语法，只需要在不同的字段使用逗号进行分隔，如下所示：
```cpp
C::C( double a, double b, double c): X(a), Y(b), Z(c) 
{ .... }
```
### 类的析构函数

类的**析构函数**是类的一种特殊的成员函数，它会在每次删除所创建的对象时执行。

析构函数的名称与类的名称是完全相同的，只是在前面加了个波浪号（~）作为前缀，它不会返回任何值，也不能带有任何参数。析构函数有助于在跳出程序（比如关闭文件、释放内存等）前释放资源。

### C++ 拷贝构造函数详解(==写的好多，跳过时要折叠==)

#### 什么是拷贝构造函数？

拷贝构造函数是一种特殊的构造函数，用于**用一个已存在的对象来初始化新创建的对象**。它的参数是对同类的引用（通常是 `const` 引用）。

#### 基本语法


```cpp

class ClassName {
public:
    // 拷贝构造函数
    ClassName(const ClassName& other) {
        // 复制 other 的数据成员
    }
};
```
#### 何时被调用？

拷贝构造函数在以下情况被自动调用：

1. **通过已有对象创建新对象**
    

```cpp

MyClass obj1;
MyClass obj2(obj1);      // 拷贝构造
MyClass obj3 = obj1;      // 拷贝构造（注意：这是初始化，不是赋值）

```
2. **对象作为函数参数按值传递**
    

```cpp

void func(MyClass obj) {  // 调用拷贝构造函数
    // ...
}
MyClass obj;
func(obj);                // 拷贝构造临时对象
```
3. **函数按值返回对象**
    


```cpp
MyClass func() {
    MyClass obj;
    return obj;           // 拷贝构造临时对象（可能被优化）
}
```
#### 默认拷贝构造函数

如果没有显式定义，编译器会生成一个默认的拷贝构造函数，它执行**浅拷贝（成员逐个复制）**：

```cpp

class Student {
    string name;
    int age;
    // 编译器生成的默认拷贝构造函数：
    // Student(const Student& other) : name(other.name), age(other.age) {}
};
```
#### 浅拷贝 vs 深拷贝

##### 浅拷贝的问题

```cpp

class StringWrapper {
    char* str;
public:
    StringWrapper(const char* s) {
        str = new char[strlen(s) + 1];
        strcpy(str, s);
    }
    
    ~StringWrapper() {
        delete[] str;  // 析构时释放内存
    }
    
    // 没有定义拷贝构造函数 → 使用默认浅拷贝
};
int main() {
    StringWrapper obj1("Hello");
    StringWrapper obj2(obj1);  // 浅拷贝：两个对象指向同一块内存
    
    // 析构时：obj2 先释放内存，obj1 再释放时出现 double delete 错误！
    return 0;
}
```

##### 深拷贝的正确实现

```cpp

class StringWrapper {
    char* str;
public:
    StringWrapper(const char* s) {
        str = new char[strlen(s) + 1];
        strcpy(str, s);
    }
    
    // 深拷贝构造函数
    StringWrapper(const StringWrapper& other) {
        str = new char[strlen(other.str) + 1];  // 分配新内存
        strcpy(str, other.str);                  // 复制内容
    }
    
    ~StringWrapper() {
        delete[] str;
    }
};
```

#### 拷贝构造函数的几种形式

##### 1. 基础形式

```cpp

class Point {
    int x, y;
public:
    Point(const Point& other) : x(other.x), y(other.y) {}
};
```

##### 2. 处理动态内存

```cpp

class Array {
    int* data;
    size_t size;
public:
    Array(const Array& other) : size(other.size) {
        data = new int[size];
        for (size_t i = 0; i < size; ++i) {
            data[i] = other.data[i];
        }
    }
};
```

##### 3. 继承体系中的拷贝构造

```cpp

class Base {
    int baseData;
public:
    Base(const Base& other) : baseData(other.baseData) {}
};
class Derived : public Base {
    int derivedData;
public:
    Derived(const Derived& other) 
        : Base(other)  // 调用基类拷贝构造函数
        , derivedData(other.derivedData) {}
};
```
#### 禁止拷贝

有时需要禁止对象被拷贝：

```cpp

class NonCopyable {
public:
    NonCopyable() = default;
    
    // C++11 及以后
    NonCopyable(const NonCopyable&) = delete;
    NonCopyable& operator=(const NonCopyable&) = delete;
    
    // C++11 之前
private:
    NonCopyable(const NonCopyable&);  // 只声明不定义
    NonCopyable& operator=(const NonCopyable&);
};
```
#### 拷贝赋值运算符的区别

不要混淆拷贝构造函数和拷贝赋值运算符：

```cpp

MyClass obj1;
MyClass obj2(obj1);     // 拷贝构造（新对象）
MyClass obj3 = obj1;     // 拷贝构造（新对象）
MyClass obj4;
obj4 = obj1;            // 拷贝赋值（已存在的对象）
```
#### 最佳实践

1. **Rule of Three**：如果定义了析构函数、拷贝构造函数或拷贝赋值运算符中的任何一个，通常需要定义所有三个。
    
2. **Rule of Five**（C++11）：加上移动构造函数和移动赋值运算符。
    
3. **使用初始化列表**，而不是在构造函数体内赋值：
    

```cpp

// 好的做法
MyClass(const MyClass& other) : data(other.data) {}
// 避免
MyClass(const MyClass& other) {
    data = other.data;  // 先默认构造，再赋值，效率较低
}
```

4. **参数使用 const 引用**，避免无限递归：
    

```cpp

// 正确
MyClass(const MyClass& other);
// 错误：会导致无限递归
MyClass(MyClass other);
```
